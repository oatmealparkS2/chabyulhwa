Type B 핵심 원칙
01페인포인트는 What으로 정의한다. "어떻게 고칠 것인가(How)"가 아닌 "무엇이 문제인가(What)"를 먼저 명확히 한다. 해결 방법은 페인포인트 정의 후에 도출한다.
02Before 수치가 먼저. 과제 검토 시점부터 해소를 측정할 수 있는 수치를 정의한다. 현재 수치를 확정하기 전까지 개선 목표 수치를 설정하지 않는다.
03측정 조건을 사전에 고정. 오픈 전후를 유효하게 비교하려면 측정 지표·기준 기간·사용자 세그먼트를 과제 시작 시점에 미리 정의해야 한다.
04Phase 5 (After 측정)은 배포 후 필수. 성과 측정 없이 과제를 종료하지 않는다. 결과(달성/부분 달성/미달성)는 최상위 문서에 기록하고 [완료] 처리한다.
