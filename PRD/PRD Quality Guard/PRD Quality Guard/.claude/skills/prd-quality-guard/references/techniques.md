# 7기법 가이드 · 기법↔요소 매핑

> 본문(아래 두 표)은 **도메인 중립** — 어떤 PRD에든 그대로 적용한다. 구체 사례가 필요하면 맨 아래 **부록 예시**를 참고하되, 그 사례를 본문 규칙으로 역수입하지 말 것(특정 도메인 앵커링 금지).

## 7기법 가이드

| 기법 | 정의 | 적용 PRD 요소(유형) | 드러내는 갭 유형 | 적용 절차 한 줄 |
|---|---|---|---|---|
| 동등분할(EP) | 입력/상태를 동작이 같은 그룹으로 나눠 대표값 1개씩 | 범주형 입력·상태·분류값 | 누락(분류 안 된 그룹) | 동작이 같은 값들을 그룹으로 묶고 각 그룹 대표값 1개씩 — 그룹 분류가 PRD에 없으면 갭 |
| 경계값(BVA) | 동등분할 경계의 직전/정각/직후 값 | 수치·비율·기간·시점이 있는 입력/조건 | 경계미정(상·하한·반올림·포함여부·시점) | 각 경계의 직전·정각·직후 3점을 TC로 — 상·하한·반올림·경계포함이 PRD에 없으면 갭 |
| 결정테이블(DT) | 조건 조합 → 행동 매핑표로 모든 규칙 열거 | 둘 이상 조건이 결합되는 정책·분기 | 모순(규칙 충돌), 누락(빈 규칙) | 조건×규칙 매트릭스를 **산출물에 출력** — 빈 동작=누락, 충돌 행=모순, 각 빈/충돌 셀을 GAP과 교차연결 (규격 ↓ "결정테이블 산출물") |
| 상태전이(ST) | 상태·이벤트·전이를 도식화 | 상태·라이프사이클을 갖는 화면·엔티티 | 누락(미정의 전이), 모호(전이 조건) | 상태×이벤트 전이표 + Mermaid 상태도를 **산출물에 출력** — 빈 칸=미정의 전이(누락)·조건 불명=모호, 각 빈 칸을 GAP과 교차연결 (규격 ↓ "상태전이 산출물") |
| 페어와이즈 | N개 요인의 값 쌍을 모두 커버하는 최소 조합 | 값이 여럿인 요인이 N개 이상 조합 | 누락(조합 폭증 속 미정의) | 모든 값 쌍을 1회 이상 덮는 최소 조합 집합 — 폭증 조합 중 동작 미정의=누락 |
| 유스케이스 | 액터의 종단 시나리오(정상+대안 흐름) | 액터의 종단 시나리오·플로우 | 누락(대안흐름), 모호(분기) | 정상 흐름 + 각 분기의 대안·예외 흐름을 끝까지 — 빠진 대안흐름=누락, 분기 기준 불명=모호 |
| 오류추측 | 경험 기반 예외·악조건 케이스 발굴 | 예외·동시성·외부장애·복구 | 누락(예외), 모호(예외 동작) | 경험상 깨지는 악조건(중복·동시·외부실패·타임아웃) 나열 — 동작 미정의=누락/모호 |

> 주: Cause-Effect Graph는 산출물이 결정테이블로 수렴하므로 DT로 흡수(별도 기법 미채택). DT의 flowchart·❓/⚠️ 셀이 CEG의 인과 시각화·조합 완전성을 대체한다.

## TC 양산 방지 규칙

TC는 두 목적을 동시에 가진다 — ① PRD 갭 발견 수단 ② 실제 QA가 실행하는 테스트 명세. 그래서 redundant TC = QA 실행 인시 낭비다. 도출 시 아래 4규칙을 강제한다(이 섹션이 단일 원본 — SKILL step3·schemas.md는 여기를 참조):

- **클래스당 대표 1개** — 동등분할은 한 동등 클래스당 TC 1개. 같은 클래스 안에서 값만 바꾼 TC는 만들지 않는다.
- **경계는 별도 경계 TC로** — 경계값(직전·정각·직후)은 클래스 대표 TC와 분리된 별도 TC. 클래스 대표와 중복시키지 않는다.
- **조합은 페어와이즈로 압축** — 다요인 조합은 모든 값 쌍을 1회 이상 덮는 최소 조합만(전수 조합 금지).
- **커버리지 매핑 의무 + 병합** — 모든 TC는 {동등클래스·경계·결정규칙·상태전이·유스케이스 경로·에러 시나리오} 중 최소 1개에 매핑된다. 두 TC가 같은 커버리지 대상에 매핑되고 추가 변별력이 없으면 병합한다.

## 기법 ↔ 요소 매핑 규칙

| PRD 요소(유형) | 1순위 기법 | 보조 기법 |
|---|---|---|
| 상태·라이프사이클을 갖는 화면·엔티티 | 상태전이 | 유스케이스 |
| 수치·비율·기간·시점 입력 | 경계값 | 동등분할 |
| 범주형 상태·분류값 | 동등분할 | 결정테이블 |
| 둘 이상 조건이 결합되는 정책 | 결정테이블 | 페어와이즈 |
| 액터의 종단 흐름 | 유스케이스 | 상태전이 |
| 예외·악조건·동시성 | 오류추측 | 결정테이블 |
| 다요인 입력 조합 | 페어와이즈 | 결정테이블 |

## 상태전이(ST) 산출물 규격

상태·라이프사이클을 가진 엔티티가 있으면, ST는 `기법` 태그로 끝내지 말고 **상태전이 모델**을 산출물(TC 파일)에 함께 출력한다. 한 모델 = (1) 상태×이벤트 전이표 + (2) 같은 모델의 Mermaid 상태도, 둘은 같은 모델의 두 표현이므로 항상 쌍으로 둔다.

- **전이표(매트릭스)** — 행=현재 상태, 열=이벤트, 셀=다음 상태/동작. 표기: 정상 전이=다음 상태명, 발생 불가=`–`, **전이 미정의/조건 모호=`❓→GAP-xx`**. 모든 상태×이벤트 칸을 채우려 시도하면 빈 칸이 곧 갭으로 드러난다.
- **Mermaid 상태도** — `stateDiagram-v2`로 같은 상태·전이를 시각화. 미정의/모호 전이는 라벨에 `❓GAP-xx` 또는 `note`로 표기. 한글 상태명은 `state "라벨" as ID` 형식으로 선언하고 라벨에 `[ ]`(start/end 예약)를 쓰지 않아 렌더 안정성을 확보한다.
- **양 산출물 배치** — 전체 전이표(매트릭스)는 TC 파일에 두고, **Mermaid 상태도는 TC 파일 + GAP 리포트 양쪽**에 싣는다. GAP 리포트는 PM이 단독으로 들고 갈 액션 리스트이므로 ST 갭을 그 자리에서 도식으로 보여줘야 한다. GAP의 상태도엔 각 `❓` 전이=GAP-ID 매핑 캡션을 달고, 양 파일 도식은 동기화한다.
- **교차연결** — 전이표의 모든 `❓` 칸은 GAP 표의 해당 행과 1:1 연결하고 관련 TC-ID도 병기한다(ST 갭의 도출 근거를 가시화). 전이가 아닌 "동작 내용"의 미정(지급 형태·금액 등)은 전이표/상태도 밑 주석으로 분리한다.
- **조건부 적용** — 단일 상태이거나 라이프사이클이 없는 요소엔 만들지 않는다(노이즈 방지). 2상태 단순 토글은 도식 생략하고 한 줄 주석으로 갈음 가능.

## 결정테이블(DT) 산출물 규격

둘 이상의 조건이 결합되는 정책·분기가 있으면, DT는 `기법` 태그로 끝내지 말고 **결정 매트릭스**를 산출물(TC 파일)에 함께 출력한다. 한 매트릭스 = (1) 조건×규칙 표가 본체이며, 조건이 3개 이상이거나 분기가 비대칭이면 (2) 같은 규칙을 Mermaid `flowchart`로 곁들인다. 단순 2×2는 표만으로 충분.

- **결정 매트릭스(조건×규칙)** — 행=조건/동작, 열=규칙(R1, R2, …). 위쪽 N행은 각 조건의 값(또는 Y/N), 아래쪽 행은 그 규칙의 `→ 동작 / → TC-ID / → GAP-ID`. 셀 표기: 정상=값 또는 `Y`/`N`, 무관=`–`, **동작 미정의=`❓→GAP-xx`**, **규칙 충돌=`⚠️→GAP-xx`**. 조건 가짓수가 작으면(≤8 조합) 완전 열거하고, 많으면 PRD에 정의된 규칙 + `R_else: 그 외`(catchall) 명시. 모든 비교집합이 한 열로 나와야 빈/충돌 셀이 곧 갭으로 드러난다.
- **Mermaid 시각화(조건부)** — 조건 ≥3개이거나 분기가 비대칭이면 `flowchart TD`로 의사결정 트리를 곁들인다. 미정의/충돌 노드 라벨에 `❓GAP-xx` / `⚠️GAP-xx` 표기. 단순 2×2 매트릭스는 도식 생략.
- **양 산출물 배치** — 전체 매트릭스는 TC 파일에 두고, **`❓`·`⚠️` 셀을 포함한 매트릭스는 GAP 리포트에도 사본 배치**(전체 또는 갭 행만 발췌). GAP 리포트는 PM 단독 액션 리스트이므로 충돌·누락 규칙을 그 자리에서 표로 보여줘야 한다. 양 파일 매트릭스는 동기화하고, 도식이 있으면 같이 미러링한다.
- **교차연결** — 매트릭스의 모든 `❓`·`⚠️` 셀은 GAP 표의 해당 행과 1:1 연결하고 관련 TC-ID도 병기한다(DT 갭의 도출 근거를 가시화).
- **조건부 적용** — 조건이 1개뿐이거나 동등분할/경계값으로 충분한 경우엔 만들지 않는다(노이즈 방지). 단일 Y/N 토글은 한 줄 주석으로 갈음.
