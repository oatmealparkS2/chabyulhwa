# 설계 — GAP 리포트 재구조화: 색인+상세블록 · 현황/문제점 분리 · 제외 처분 · Confluence 링크

> 날짜: 2026-06-02
> 대상: 프로젝트 로컬 스킬 `prd-quality-guard` (산출물 스키마 + 검증 + 첫 인스턴스 retrofit)
> 분류: PM 소비/워크플로 개선 (갭 발견 로직 불변 — 산출물 표현·처분 레이어만 변경)

## 배경 / 문제

세 가지 사용자 요청에서 출발:

1. **(R1)** GAP 리포트 `설명` 칸이 "사실 서술"과 "왜 갭인가"를 한 문장에 뭉쳐 너무 길다 → **현황 / 문제점**으로 분리.
2. **(R2)** PM이 특정 갭을 의도적으로 반영 안 하기로 할 때(예: GAP-01) **제외 + 사유를 기록**할 수단이 없다.
3. **(R3)** TC·GAP의 `위치` 약기호(`L2 2-1·OI-01 ↔ L4 §2·5`)가 사람이 보기 힘들다 → **실제 Confluence 링크 병기**.

공통 관통 사실: 현행 **8열 갭 표가 포화**다. 세 요청 모두 한 행에 정보를 더 얹는 것이라, 표 한 칸에 욱여넣는 대신 **색인 표 + GAP별 상세블록**으로 분리하는 것이 셋을 동시에 자연스럽게 수용한다. 이 셋은 갭 검출(엔진)을 바꾸지 않는 **PM 소비/처분 레이어** 개선이다.

## 결정 요약 (확정된 설계 선택)

| 항목 | 결정 |
|---|---|
| 리포트 구조 | 색인 표(6열) + GAP별 상세블록 + 기존 ST/DT 도식 섹션 유지 (3단) |
| R2 제외 영속화 | GAP 파일 내 `처분` 필드 + 결번 색인 carry-forward (기존 delta 레일 확장) |
| R3 링크 입도 | **페이지 단위** (섹션 딥링크는 신뢰성 없어 포기) |
| 인스턴스 retrofit | 이번에 스킬과 **함께** (과거 스키마 변경 소급 패턴) |

## 설계

### A. 새 GAP 리포트 레이아웃 (3단)

**1단 — 색인 표 (심각도순, 스캔용) · 6열**

| GAP-ID | 위치 | 기법 | 분류 | 심각도 | 한줄요약 |
|---|---|---|---|---|---|

- High N건 한눈 조망 유지. 뚱뚱한 `설명`/`PM질문/제안`은 표에서 제거 → 상세블록으로.
- `위치`는 링크 포함(C 참조). `한줄요약` = 갭 1줄 압축.

**2단 — GAP별 상세블록**

```
### GAP-01  모순·High·상태전이
- 위치: [L2 2-1·OI-01](url) ↔ [L4 §2·5](url)
- 현황: (사실 서술 — PRD가 무엇을 말하는가)
- 문제점: (왜 갭인가 — 어떤 리스크/공백)
- PM 질문/제안: …
- 처분: ☑유지(기본)  ☐제외(사유: ___)  ☐오판정(사유: ___)
- 관련 TC: US04-T04 → (TC 파일 참조)
```

- 헤딩 = `### GAP-ID  분류·심각도·기법` (색인 표와 분류/심각도/기법 동기).
- 색인 표의 모든 GAP-ID ↔ 상세블록 **1:1**.

**3단 — 기존 ST/DT 도식 섹션 유지**

- 현 §2 상태전이 도식·§3 결정 매트릭스 사본은 *여러 GAP을 가로지르는 엔티티 모델*이라 상세블록에 흡수하지 않고 **독립 섹션으로 유지**. 상세블록에서 `→ M2 도식 참조`로 교차링크.

### B. R1 — 현황 / 문제점 분리

상세블록의 `현황`·`문제점`을 **별도 필드로 강제 분리**. 산문 뭉치기 금지.
- `현황` = PRD가 실제로 말하는 사실(예: "L2 리워드 10,000원 쿠폰 확정, OI-01 완료").
- `문제점` = 왜 갭인가 / 리스크(예: "L4 TechSpec은 여전히 [TBD] — 배치가 두 값을 가리킴").

### C. R3 — Confluence 링크 (TC·GAP 양쪽)

- **입도 = 페이지 단위.** 섹션 딥링크(2-1·OI-01 정확한 헤딩)는 Confluence 자동 생성 앵커가 취약 + 스킬이 MCP markdown으로 페치해 안정적 앵커맵이 없어 **신뢰성 없음** → 포기. 섹션 약기호(`2-1·OI-01 §2·5`)는 링크 텍스트 안에 그대로 둬 페이지 내 위치표시 역할.
- **L#→URL 매핑 원천 = 헤더 메타 블록.** `대상 PRD` 줄에 L# 라벨을 명시:
  `> **대상 PRD**: L2 · pageId 2083160088 · v.. · lastModified .. · [링크](url)`
  위치의 L-토큰이 이 맵에 해소돼 하이퍼링크된다. L# 배정은 분석 중 엔진이 수행(페이지 역할 기반).
- TC 파일도 동일 적용(위치/전제조건의 L-토큰 링크).

### D. R2 — 제외 처분 (in-cycle 기록 + 재실행 carry-forward)

**처분 필드 = 3상태로 통일** — 기존 결번색인의 `철회(오판정)` 슬롯에 in-cycle 진입점도 동시 부여:
```
- 처분: ☑유지(기본)  ☐제외 — 갭은 실재하나 미반영(의도된 동작/범위 외/리스크 수용) · 사유: ___
                      ☐오판정 — 엔진 오검출(가짜 갭) · 사유: ___
```
`제외`와 `오판정`은 둘 다 "메인에서 빠지지만 사유를 남긴다"는 점은 같고, **왜 빠지는지가 정반대**(PM이 맞다고 보지만 안 함 vs 엔진이 틀림)라 라벨을 구분해 기록을 정직하게 만든다.

**1회차:** PM이 체크+사유를 적으면 그 자체로 기록 완료(R2 핵심 충족). 재실행이 없어도 OK.

**재실행 carry-forward (SKILL.md step5 delta 로직 확장):** 엔진은 이미 이전 GAP 파일을 읽는다. 한 단계 추가:
1. 이전 GAP 파일 상세블록에서 `☑제외`/`☑오판정` + 사유를 **수확**.
2. 해당 GAP-ID를 새 색인·상세블록에서 **억제**(PRD에 갭이 남아 있어도 재등장 금지 — 좀비 방지).
3. 결번 색인에 이월: `GAP-xx: [제외 YYYY-MM-DD · pageId v..] 사유` / `[오판정 …] 사유`.
4. 변경이력에 신규 마커 `🚫 제외: GAP-xx (사유)` (기존 ✅해결/🔁재발/🆕신규/➖잔존에 추가).

**제외 철회(PM 마음 변경):** 기존 `🔁재발` 레일 재사용 — 결번색인 stub 제거 → 다음 회차 메인 복귀.

**GAP-ID 항구성과의 정합:** 제외/오판정 = 해결과 동일하게 메인에서 빠지되 결번색인에 stub 유지. 신규는 여전히 max+1(결번 안 메움). 재발 예외도 그대로.

### E. 검증 (verification.md) 변경

- **C2 재작성** (현재 8열 표 하드코딩 → 신구조 정합):
  1. 색인 표 6열(`GAP-ID·위치·기법·분류·심각도·한줄요약`) 존재.
  2. 색인의 모든 GAP-ID에 대응 상세블록 **1:1** 존재(누락·고아 블록 = FAIL).
  3. 상세블록 필수 필드 `위치·현황·문제점·PM질문/제안`이 모두 비어있지 않음.
  4. 분류·심각도는 상세블록 헤딩에서 확인.
- **C5 보강**: 결번색인 stub 종류에 `제외·오판정` 추가 인식. 제외/오판정 ID는 메인(색인·상세블록) 재등장 금지(재발 예외 유지).
- **C7 보강**: 변경이력 `🚫제외` 항목 ↔ 결번색인 stub(사유 포함) 정합 확인. 처분 수확 정합.
- **검증 범위 밖(불변)**: 링크 정확성(앵커 해소)은 검증자가 외부 페치 금지라 구조 검증 대상 아님 — `위치` 비어있음만 본다(현행 C2 정신 유지).

### F. 영향 파일 + 인스턴스 retrofit

**스킬 단일원본 변경:**
- `references/schemas.md` — 갭 리포트 표 템플릿 → 3단 템플릿(색인 표 6열 + 상세블록 템플릿 + 처분 필드 + 결번색인 taxonomy 확장 + 헤더 L# 라벨).
- `SKILL.md` — step1(L# 라벨 발행)·step5(신 레이아웃 + 처분 수확 carry-forward + L# 헤더).
- `references/verification.md` — C2 재작성, C5/C7 보강.
- `references/techniques.md` — 소폭(상세블록 교차링크 언급) 또는 무변.
- `_template/CLAUDE.md` — 소폭(PM 처분 사용 안내) 또는 무변.

**인스턴스 retrofit (차별화리뷰Pro-추천인코드):**
- GAP 파일: 24건을 3단 레이아웃으로 전환 — 설명→현황/문제점 분리, 처분=전부 유지 기본, 위치 페이지 링크, 색인 표 6열 생성, 헤더 L# 라벨.
- TC 파일: 위치 L-토큰 링크.
- **이번 작업 노동의 대부분.**

## 비목표 (scope 박음)

- 갭 검출 로직·7기법·양산방지·ST/DT 산출 규격 **불변**. 이번 변경은 표현·처분 레이어 한정.
- 섹션 단위 딥링크 **안 함**(신뢰성 없음).
- 우선순위 열 **여전히 안 둠**(기존 비목표 유지 — QA 다운스트림).
- 링크 정확성 자동검증 **안 함**.

## 검증 계획

- 변경 후 독립 구조검증 subAgent(general-purpose·sonnet)를 retrofit된 인스턴스 2파일에 재디스패치 → **C1~C7 PASS** 확인(C7은 인스턴스가 1회차라 N/A).
- 결함 주입 회귀: 상세블록 1:1 위반(C2)·결번색인 제외 stub 누락(C5/C7)을 주입해 FAIL 정확 지목 확인.
- mermaid 도식은 변경 없음(기존 6도식 렌더 검증 유지).

## 미해결 / 후속

- 회차/버전 관리 실회차 시연(Task12)에서 처분 carry-forward를 실증할 수 있으면 더 좋다(별도 작업).
