# 설계 — PRD 품질 가드 자체검증의 subAgent 독립화

- 작성일: 2026-05-27
- 출력 언어: 한국어
- 대상 스킬: `PRD Quality Guard/.claude/skills/prd-quality-guard/` (프로젝트 로컬)
- 선행 설계: `2026-05-27-skillify-prd-quality-guard-design.md` (§7 자체검증)

## 1. 목적 & 동기

현재 `SKILL.md`의 **6. 자체검증**은 TC/GAP를 *생성한 바로 그 세션*이 자기 산출물을 5개 체크리스트로 점검한다. 같은 컨텍스트가 자기 결과를 검사하므로 **자기확신 편향**(생성 근거를 정답으로 신뢰)이 끼어든다.

이를 **격리된 fresh-context subAgent**가 검증하도록 바꾼다. 생성 근거를 모르는 검증자는 "쓰여 있는 것"만으로 판정하므로 편향이 제거된다.

## 2. 확정된 결정사항 (브레인스토밍 합의)

| 항목 | 결정 |
|---|---|
| 검증 깊이 | **구조 정합성만**. PRD 재페치 없이 TC/GAP 두 파일만 읽어 기존 5체크를 기계적으로 판정. |
| 트리거 | **생성 직후 자동**. step5에서 파일 작성 직후, 생성 세션이 Agent 도구로 검증 subAgent를 디스패치. |
| FAIL 처리 | **보고만**. verdict를 가공 없이 사용자에게 전달. 자동 수정 없음(사용자 판단). |
| 구현 형태 | **접근 A** — 새 reference 파일에 자기완결형 검증 프롬프트를 두고 SKILL.md step6은 디스패치 절차로 재작성. |
| subagent 종류 | **general-purpose** (model: sonnet). Explore는 excerpt만 읽고 audit 안 하므로 부적합. |
| verdict 산출물 | **인라인 전달, 3번째 파일 없음**. 기존 "산출물 2파일" 규칙 보존. |

## 3. 아키텍처 개요

```
생성 세션 (prd-quality-guard 스킬 실행)
  step5: doc/TC-<PRD명>.md + doc/GAP-<PRD명>.md 작성
  step6: references/verification.md 프롬프트에 두 파일 경로 주입
         → Agent 도구로 검증 subAgent 디스패치 (general-purpose, sonnet)
                                   │
                                   ▼
            검증 subAgent (격리된 fresh context = "새 세션")
              - 입력: TC 파일·GAP 파일 절대경로 2개 (PRD 재페치 금지)
              - 5개 구조 체크를 증거와 함께 판정
              - 전체 verdict + 체크별 결과 반환 (파일 안 씀)
                                   │
                                   ▼
  step6 (이어서): 생성 세션이 verdict를 가공 없이 사용자에게 전달
                  (FAIL이어도 자동 수정 안 함)
```

**독립성 보장**: Agent 도구로 띄운 subagent는 생성 세션과 분리된 fresh context를 가진다 → 생성 추론을 볼 수 없음 = "새 세션" 요건 충족. 검증 프롬프트는 자기완결형이라 subAgent가 스킬을 로드할 필요도 없다.

## 4. 구성요소 상세

### 4.1 새 파일 `references/verification.md`

검증 subAgent에 **그대로 넘길 자기완결형 프롬프트**. 구성:

**(a) 역할 선언**
> 너는 PRD 품질가드 산출물의 *독립 구조검증자*다. 생성 근거를 신뢰하지 말고, 넘겨받은 두 파일에 *쓰여 있는 것만으로* 판정하라. PRD를 다시 페치하지 마라(구조 정합성만 본다). 모든 판정에는 증거(개수·ID·줄)를 붙여라. 추측하지 말고, 파일에 근거가 없으면 FAIL로 보고하라.

**(b) 입력 규약**
- 디스패치 시 주입되는 두 절대경로: TC 파일, GAP 파일.
- 그 외 어떤 외부 소스도 참조 금지(특히 Confluence PRD).

**(c) 5개 체크 + 증거 규칙**

| # | 체크 | PASS 판정 근거 / FAIL 시 보고 내용 |
|---|------|-----------------------------------|
| C1 | 커버리지 누락 0 | 매트릭스 요소군 행 전체 나열, 셀이 비어있는(TC·갭 모두 0) 요소군 ID 목록 |
| C2 | 모든 갭 4필드(위치·분류·심각도·제안) 충족 | 4필드 중 하나라도 결손인 GAP-ID 목록 |
| C3 | 7기법 각각 ≥1 TC에서 행사 | 기법별 등장 TC 수, 등장 0인 기법명 목록 |
| C4 | 상태·라이프사이클 엔티티마다 ST모델(전이표+Mermaid 상태도) 존재 & 모든 `❓` 칸이 GAP과 연결 | 엔티티별 전이표·상태도 유무, GAP 미연결 `❓` 칸 위치 목록 |
| C5 | TC-ID/GAP-ID 유일·일관 & TC↔GAP 상호참조 유효 | 중복·결번 ID, 가리키는 대상이 없는 dangling 참조(`→ GAP-xx` 등) 목록 |

**(d) 출력 형식**
```
## 검증 결과: PASS | FAIL
- C1 커버리지: PASS/FAIL — <증거>
- C2 갭 4필드: PASS/FAIL — <증거>
- C3 7기법 행사: PASS/FAIL — <증거>
- C4 ST모델·❓연결: PASS/FAIL — <증거>
- C5 ID·상호참조: PASS/FAIL — <증거>
(FAIL 항목은 위반 목록을 빠짐없이 나열)
```
파일에 쓰지 않고 디스패처에 텍스트로 반환한다.

### 4.2 `SKILL.md` step6 재작성

현재의 5줄 자가검증 체크리스트를 **디스패치 절차**로 교체:

> ### 6. 자체검증 (독립 subAgent)
> step5 완료 후, 생성 세션은 `references/verification.md`의 프롬프트 본문에 두 산출물의 절대경로(TC 파일·GAP 파일)를 주입해 **검증 subAgent를 Agent 도구로 디스패치**한다(subagent_type: general-purpose, model: sonnet). 반환된 verdict를 **가공 없이 사용자에게 전달**한다. FAIL이어도 자동 수정하지 않는다(수정 여부는 사용자 판단). 검증자는 PRD를 재페치하지 않고 두 파일의 **구조 정합성만** 본다.

5개 체크의 *정의*는 step6 본문에서 `references/verification.md`로 이전(중복 방지). step6은 "무엇을 검사하는지"는 verification.md를 참조한다고만 기술.

## 5. 데이터 흐름 / 계약

```
step5 작성 완료 → 생성 세션이 두 파일 절대경로 확보
   → verification.md 프롬프트 + 경로 주입 → Agent(general-purpose, sonnet) 디스패치
검증 subAgent: 두 파일 읽기 → C1~C5 판정(증거 첨부) → verdict 반환
생성 세션: verdict 인라인 전달 (수정 없음)
```

**계약(불변)**: 검증 subAgent는 *주입된 두 파일 경로*만으로 동작하며 외부 소스를 참조하지 않는다. 이 격리가 독립성과 결정성을 동시에 보장한다.

## 6. 자체검증 / 테스트 (이 변경의 검증)

1. **트리거**: 생성 세션이 step5 직후 검증 subAgent를 실제로 디스패치하는가.
2. **격리**: subAgent가 PRD를 재페치하지 않고 두 파일만 읽는가(프롬프트가 이를 강제).
3. **민감도**: 일부러 깨뜨린 산출물(예: GAP 한 건의 심각도 칸 삭제, dangling `→ GAP-99` 삽입, 7기법 중 하나 제거)에 대해 검증자가 해당 체크를 **FAIL + 정확한 위반 항목**으로 잡아내는가.
4. **무결 케이스**: 현재 첫 시연 인스턴스(TC 42 / GAP 24) 산출물에 돌렸을 때 **PASS** 나오는가.
5. **보고**: verdict가 가공 없이 사용자에게 전달되고 자동 수정이 일어나지 않는가.

## 7. 범위 밖 (YAGNI)

- **독립 재분석**(PRD 재페치해 누락 TC·갭 헌팅) — 이번엔 구조 정합성만. 추후 별도 옵션.
- **자동 수정 루프**(FAIL → 고쳐서 재검증) — 보고만으로 확정. 추후 옵션.
- **verdict 영속 파일**(`doc/VERIFY-*.md`) — 2파일 규칙 보존 위해 미채택.
- **커스텀 에이전트 정의**(`.claude/agents/prd-verifier`) — 스킬 자기완결성 위해 미채택(접근 C 기각).
