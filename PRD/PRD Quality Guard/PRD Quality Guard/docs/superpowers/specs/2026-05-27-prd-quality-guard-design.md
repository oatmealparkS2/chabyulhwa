# 설계 — PRD 품질 가드: 블랙박스 TC 기반 갭 분석

- 작성일: 2026-05-27
- 출력 언어: 한국어
- 대상 PRD: 차별화리뷰 Pro 추천인코드 적용 (Confluence, cloudId `smartfoodnet.atlassian.net`)

## 1. 목적 & 핵심 원칙

블랙박스 테스팅 7기법으로 PRD에서 테스트 케이스(TC)를 도출하고, **TC 도출이 막히는 지점을 PRD 갭으로 기록**한다.

> **핵심 원칙: "완전한 TC를 못 쓰면 = PRD 갭."**

TC 생성 과정을 강제 함수(forcing function)로 사용해, 사람이 그냥 읽어서는 놓치는 PRD의 모순·누락·모호·경계미정을 체계적으로 드러내는 것이 목표다.

## 2. 산출물 형태 & 방향

- **형태**: 한국어 마크다운 문서 1개.
- **방향 A (둘 다)**: 1부에 재사용 방법론, 2부에 이 PRD를 worked example로 적용. 방법론을 실제 PRD로 시연하는 구조.
- **TC 역할 = 균형**: 기법/요소별 대표 TC는 완전한 표로, 나머지는 갭 도출이 보이는 스케치로. **갭 리포트가 주연, TC는 시연 + 증거.**

## 3. 문서 전체 구조

```
Part 0  개요         목적 / 핵심원칙 / 대상 PRD 범위(L1~L4) / 사용법
Part 1  재사용 방법론  (범용 — 어떤 PRD에도 적용)
Part 2  이 PRD 적용    (Part 1을 차별화리뷰Pro PRD로 시연, 하이브리드 구성)
부록    종합 갭 리포트  (모든 갭 1표 집계, 심각도순 — PM 단독 액션 리스트)
```

- Part 1 = "재사용" 목표, Part 2 = "즉시 가치(실제 갭)", 부록 = "PM이 들고 갈 결과물".

## 4. 블랙박스 7기법

동등분할(EP) · 경계값(BVA) · 결정테이블(DT) · 상태전이(ST) · 페어와이즈 · 유스케이스 · 오류추측.

각 기법을 PRD 요소(US/AC · 화면상태 · 정책 · 계산로직 · 비기능)에 적용해 TC를 도출하고, 도출이 막히는 지점을 갭으로 기록한다.

## 5. Part 1 — 재사용 방법론 (범용)

1. **프로세스 5단계**: PRD 입력 → 요소 추출(US/AC·화면상태·정책·계산로직·비기능) → 요소별 7기법 매핑 → TC 도출 시도 → 막히면 갭 기록.
2. **7기법 가이드**: 기법마다 〔정의 / PRD의 어떤 요소에 / 드러내는 갭 유형 / 적용 한 줄 예〕.
3. **기법↔요소 매핑 규칙**: 어떤 요소엔 어떤 기법이 1순위인지 (예: 화면상태→상태전이, 입력필드→경계값, 정책 조합→결정테이블).
4. **갭 분류 체계**: 모순 / 누락 / 모호 / 경계미정 / TBD — 각 정의 + 판별 기준.
5. **심각도 기준**: High(배포 차단) / Med(합의 필요) / Low(문구 수준).
6. **빈 템플릿 3종**: TC 표 · 갭 리포트 표 · 커버리지 매트릭스.
7. **Category 어휘 도출법**: PRD의 기능 분해 단위(유저스토리 묶음 / 화면군 / 정책 영역)에서 5~10개의 상호배타적 기능 버킷을 추출한다. 화면 URL 그룹·US 제목·정책 섹션이 1차 후보. (값 목록은 PRD별로 달라지므로 Part 1엔 규칙만, 값은 Part 2에서 인스턴스화.)

## 6. Part 2 — 이 PRD 적용 (하이브리드)

- **2.0 커버리지 매트릭스**: 행 = PRD 요소(US-01~07, 가입지면 화면군, BO 화면군, 정책 §2-1~2-5, 계산로직, 비기능) × 열 = 7기법 → 셀에 〔적용여부·TC수·갭수〕.
- **2.1~ 요소별 섹션** (요소마다): 적용 기법 → 대표 TC 완전표 → 추가 TC 스케치 → 발견된 갭 행.

본문은 PRD 요소별로 묶어 추적성을 확보하고, 각 TC/갭 행에 사용 기법을 태그한다. 맨 앞 커버리지 매트릭스로 7기법이 모두 행사됐는지 한눈에 보인다.

## 7. 스키마 (고정)

### 7-1. TC 스키마 (7열)

`TC-ID · Category · Summary · 전제조건 · 단계 · 기대결과 · 적용기법`

| 필드 | 설명 |
|---|---|
| TC-ID | `US01-T01` 형식 (요소 + 일련번호) |
| Category | 기능별 분류 (통제 어휘, §7-3) |
| Summary | 무엇을 테스트하는지 한 줄 요약 |
| 전제조건 | 사전 상태/데이터 |
| 단계 | 실행 단계 |
| 기대결과 | 예상 결과 |
| 적용기법 | 도출에 쓴 블랙박스 기법 |

### 7-2. 갭 리포트 스키마 (8열)

`GAP-ID · 위치 · 관련TC · 기법 · 분류 · 심각도 · 설명 · PM질문/제안`

| 필드 | 설명 |
|---|---|
| GAP-ID | `GAP-01` |
| 위치 | 문서 § (예: L2 §2-5 ↔ L3-1 S1-B) |
| 관련 TC | 막힌 TC-ID |
| 기법 | 갭을 드러낸 기법 |
| 분류 | 모순 / 누락 / 모호 / 경계미정 / TBD |
| 심각도 | High / Med / Low |
| 설명 | 갭 내용 |
| PM 질문/제안 | 해소를 위한 질문 또는 제안 |

### 7-3. Category 통제 어휘 — 이 PRD 인스턴스 (8개)

| Category 값 | 포함 범위 |
|---|---|
| 코드유효성검증 | 코드 존재/만료/이력/이미사용 판정 (US-01) |
| 코드저장·승계 | localStorage 24h, 비회원 가입 후 승계, 복수코드 (US-02·03) |
| 할인적용·계산 | N개월 자동차감, 1원 엣지, 기간종료 (US-07, L4 §6) |
| 리워드지급 | 추천인/가입자 리워드 트리거·환불 (US-04) |
| 가입화면UI | 가입지면 상태 S1~S4 렌더링 |
| BO설정 | 추천코드 등록/수정, 유효성 (US-05) |
| BO조회 | 회원관리·결제내역·회원상세 (US-06) |
| 비기능 | 성능 3초, 동시결제, API오류 (L2 §2-5) |

(다른 PRD에 적용 시 §5-7 도출법으로 새 값 집합을 만든다.)

## 8. 범위 & 자체검증

- **범위 = 전체 스윕**: US-01~07 전부 · 화면상태 약 28개 전부(가입지면 11개: S1 4 + S2 3 + S3 1 + S4 3 / BO 17개: S1 4 + S2 2 + S3 3 + S4 2 + S5 6) · 정책 §2-1~2-5 · 계산로직(L4 §6) · 비기능(L2 §2-5). 이미 알려진 TBD(OI-01·02 등)도 갭으로 편입하되 **"기인지(known)"** 태그로 구분.
- **자체검증 체크**:
  1. 모든 US/화면상태가 커버리지 매트릭스에 1회 이상 등장(커버리지 누락 0).
  2. 모든 갭이 위치·분류·심각도·제안 4필드를 채움.
  3. 기법별 대표 TC 최소 1개.
