# ADR-001: Cause-Effect Graph를 결정테이블(DT)로 흡수

- 작성일: 2026-06-01
- 출력 언어: 한국어
- 대상 스킬: `PRD Quality Guard/.claude/skills/prd-quality-guard/` (프로젝트 로컬)
- 관련 파일: `references/techniques.md` (7기법 표 + 하단 각주)
- 유형: ADR (Architecture Decision Record) — "왜 7기법에 CEG가 없는가"의 결정 기록

> 이 프로젝트의 `docs/superpowers/specs/` 문서들은 ADR 역할을 겸한다(루트 CLAUDE.md: spec/plan = ADR records). 본 문서는 기존 `*-design.md`(다단계 변경 설계)와 달리 **단일 설계 결정 1건**을 정통 ADR 4칸(Status / Context / Decision / Consequences)으로 박제한다.

## Status

**Accepted** (2026-06-01)

## Context

PRD 품질 가드는 블랙박스 테스트 기법으로 TC를 도출하며 갭을 찾는다. 채택 기법은 7개 — 동등분할(EP)·경계값(BVA)·결정테이블(DT)·상태전이(ST)·페어와이즈·유스케이스·오류추측.

표준 블랙박스 기법 목록에는 **Cause-Effect Graph(CEG, 원인-결과 그래프)** 도 포함되는데 우리 7기법에는 없다. 7기법 표만 보면 이 부재가 "의도된 배제"인지 "누락"인지 구분되지 않는다 — 실제로 이 의문이 사용자로부터 제기되었고, 같은 질문이 미래에도 재발할 수 있다.

CEG의 표준 절차:

```
원인(입력)·결과(출력) 식별
  → 불린 논리 그래프(AND/OR/NOT) 작성   ← CEG 고유 단계
  → 결정테이블로 변환                    ← 실제 TC가 여기서 나옴
  → TC 도출
```

핵심 관찰: **CEG의 최종 산출물 = 결정테이블이다.** 그래프는 결정테이블에 도달하기 위한 그래픽 중간 표현이며, 실제 테스트 도출은 변환된 결정테이블에서 이루어진다.

이 방법론의 1차 목적은 TC 자체가 아니라 **"완전한 TC를 못 쓰는 지점 = PRD 갭"** 의 발견이다. 따라서 기법 채택의 기준은 "표준 목록에 있는가"가 아니라 "갭 발견·QA 실행에 고유하게 기여하는가"다.

## Decision

**Cause-Effect Graph를 독립 기법으로 채택하지 않고, 결정테이블(DT)로 흡수한다.**

근거:

1. **산출물 수렴** — CEG를 돌려도 결과물은 결정테이블이다. 종착지(DT)를 1급 기법으로 직접 채택하고, 그래프 중간 단계를 생략한다. 두 기법을 병렬로 두면 같은 갭을 이중 라벨링하는 잡음만 생긴다.

2. **갭 노출 메커니즘 동등** — CEG의 가치는 원인-결과 조합을 빠짐없이 강제하는 조합 완전성이다. 이는 DT 매트릭스의 조건×규칙 전수 열거 + `❓`(동작 미정의=누락)·`⚠️`(규칙 충돌=모순) 셀로 이미 실현된다.

3. **시각적 인과성 복원** — CEG의 유일한 추가 가치인 "인과의 시각화"는, DT 규격에서 조건 ≥3개거나 분기가 비대칭일 때 Mermaid `flowchart`를 곁들이도록 강화하여 흡수했다.

4. **매체 적합성** — CEG의 불린 그래프 노테이션은 텍스트/Confluence 산출물에서 유지보수가 어렵고 규모가 커지면 무너진다. 표 형태인 DT가 PRD 갭 리포트 매체에 더 맞는다.

흡수 명시 범위는 **CEG→DT 1건으로 한정**한다. 다른 표준 기법(실험설계법/orthogonal array, 신택스 테스팅 등)의 부재까지 일일이 변명하지 않는다 — 흡수 관계가 명확한 건만 기록한다.

## Consequences

**긍정**

- "왜 CEG가 없냐"는 재발 질문을 차단(ADR 효과). `techniques.md` 각주 1줄이 입구를 막고, 본 ADR이 전체 추론을 보존.
- DT 강화 장치(`flowchart`, `❓/⚠️` 셀)가 단순 부가 기능이 아니라 **CEG 흡수 설계의 일부**임을 명시 → 미래에 DT를 단순화하려는 시도가 이 맥락을 보고 멈출 수 있다.
- 방법론이 "임의 7개 선택"이 아니라 "표준 기법 검토 후 흡수/채택 판단"이라는 신뢰 신호.

**부정·비용**

- `techniques.md`는 스킬 실행 시 매번 컨텍스트에 적재되는 활성 파일 → 설계 서사를 본문에 넣으면 갭 도출과 무관한 토큰이 매 실행마다 실린다. **완화**: 활성 파일에는 각주 1줄만, 전체 추론은 본 ADR(비적재 문서)로 분리.
- 선례 개방 리스크 — CEG를 명시하면 다른 기법 부재 질문이 따라올 수 있다. **완화**: 흡수 관계 명확한 1건만 기록하는 선을 Decision에 못박음.

**후속 신호 (revisit 조건)**

- 특정 PRD에서 "원인→결과 추적의 명시성"이 DT만으로 부족하다고 판단되면, 별도 기법(CEG) 부활이 아니라 **DT 조건 컬럼을 원인 단위로 더 잘게 분해**하는 것으로 보강한다. 이 ADR을 뒤집을 사유는 아니다.
