# 설계 — GAP 리포트 v2: 섹션 딥링크 + 하이브리드 상세표

> 작성일 2026-06-02 · 상태: 설계 승인 대기 → 구현 예정
> 선행: `2026-06-02-gap-report-restructure-disposition-links-design.md`(3단 구조·처분·L#링크 도입). 본 스펙은 그 위에 두 가지 개선을 얹는다.

## 1. 배경 / 문제

PM(오너) 피드백 2건:

1. **위치 링크가 페이지 단위뿐.** 예) GAP-01의 `[L2 2-1·OI-01]`이 `.../pages/2082799647`(페이지 상단)로만 연결. 기대는 섹션 딥링크 `.../pages/2082799647/01.+_+Pro#2-1.-%EB%A6%AC%EC%9B%8C%EB%93%9C-%EC%B2%B4%EA%B3%84`. — 과거(06-02 1차)에 "앵커 불안정"을 이유로 미채택했으나, 실측으로 앵커가 결정론적으로 도출 가능함을 확인 → 재채택.
2. **GAP 상세블록을 표 형태로.** 현재 불릿 리스트. 구조 균일성·스캔성을 위해 표로.

## 2. 결정 (검토 결과)

### 2.1 섹션 딥링크 — 채택

**앵커 도출 규칙 (실측 확정, 2026-06-02):**

페이지 2082799647의 실제 헤딩 `### 2-1. 리워드 체계` ↔ PM 기대 앵커 `#2-1.-%EB%A6%AC%EC%9B%8C%EB%93%9C-%EC%B2%B4%EA%B3%84`(= `#2-1.-리워드-체계`) 대조 결과:

```
anchor = 헤딩텍스트의 공백 → "-" 치환, 비ASCII는 UTF-8 퍼센트인코딩, 점(.) 보존
deeplink = <풀URL(타이틀 슬러그 포함)>#<anchor>
```

규칙 확정 근거: 위 두 문자열이 정확히 일치(URL-decode 시 동일).

**엔진 동작:**

1. **L#→URL 맵 = 풀 URL(슬러그 포함) 보존.** 현재 엔진은 슬러그를 떼고 `/pages/<id>`만 base로 쓴다. CLAUDE.md "대상 PRD"의 풀 URL을 그대로 base로 유지한다(프래그먼트가 안정적으로 동작하려면 슬러그 포함 정규형 필요; 슬러그 문자열은 입력 URL 그대로 사용 — Confluence가 pageId로 정규화·리다이렉트하므로 슬러그 형태는 관대).
2. **step2 페치 시 페이지별 헤딩 인벤토리 수집.** 본문은 이미 MCP로 읽으므로 추가 페치 0. `{헤딩텍스트 → anchor}` 맵 구성(헤딩 마크다운 `#~######` 라인 추출 → 규칙 적용).
3. **위치 토큰 해소:**
   - 헤딩인 토큰(예 `2-1`, `2-5`, `US-01`) → 자기 헤딩 앵커.
   - 표 행/하위항목 토큰(예 `AC-4`, `OI-01`, `§6`, `S1-B`) → **가장 가까운 상위 헤딩 앵커**에 태움.
   - 매칭 불확실 → **페이지 레벨 폴백**(슬러그 포함 풀URL, 프래그먼트 없음).
4. **취약성 캡션(채택):** 앵커는 분석 version에 고정. 헤딩 rename 시 앵커가 깨지고 페이지 상단으로 폴백(치명적 아님·재실행 시 재생성). GAP 파일 헤더 메타에 한 줄 명기.

### 2.2 GAP 상세 = 하이브리드 표 — 채택

서술 필드는 세로 2열 표, `처분`만 표 밖 편집줄로 분리:

```
### GAP-NN  〈분류〉·〈심각도〉·〈기법〉
| 항목 | 내용 |
|---|---|
| 위치 | [L# 약기호](base#anchor) … |
| 현황 | … (다중 문장은 <br>로 줄바꿈) |
| 문제점 | … |
| PM 질문/제안 | … |
| 관련 TC | 〈TC-ID〉 · 도식 있으면 → M2 도식 참조 |

처분: ☑유지(기본)  ☐제외(사유: ___)  ☐오판정(사유: ___)
```

근거:
- **`처분`을 표 밖 한 줄로 두는 이유** = PM이 직접 체크/사유 입력하는 편집 대상. 표 셀 편집은 파이프 정렬이 깨져 불편. 또한 step5.1.5 carry-forward 수확이 `☑제외`/`☑오판정` 라인을 파싱하므로 한 줄 유지가 안전.
- 셀 내 다중 문장(①② 열거 포함)은 `<br>`로 줄바꿈(마크다운 표 관용구·렌더 정상).
- 채택 거부안: **전체 1표(행=GAP×열=필드)** = 06-02 1차에서 버린 "뚱뚱한 8열 표"의 회귀(산문 셀 폭발) → 비채택.

## 3. 변경 범위

### 3.1 엔진 (3파일)

**`SKILL.md`**
- step1: "(섹션 딥링크는 앵커 불안정으로 안 함)" 문구 제거 → 풀URL base 보존 + 딥링크 발행 + 앵커 도출 규칙 + 폴백 + 취약성 캡션 지시.
- step2: 페치 시 페이지별 헤딩 인벤토리 캡처 1줄 추가.
- step5: GAP 상세블록 서술 = 하이브리드 표; 위치 토큰 = 딥링크(폴백 포함). TC 파일도 헤더 base 슬러그 + 본문 기존 위치 하이퍼링크 딥링크화.

**`references/schemas.md`**
- "앵커 도출 규칙" 소절 신설(2.1 규칙 + 폴백).
- 2단 상세블록 템플릿(현 line 43~49) → 하이브리드 표 형태로 교체(`처분` 분리줄 + `<br>` 주석).
- 위치 필드 주석·헤더 메타 주석(현 line 44·115)을 "페이지 단위" → "섹션 딥링크(폴백 시 페이지)"로 갱신. 헤더 메타 `대상 PRD` 줄 = 풀URL(슬러그) base.
- 헤더 메타 블록에 앵커 취약성 캡션 한 줄 추가.

**`references/verification.md`**
- C2: 상세블록 필수필드 탐지를 불릿(`- 위치:`) → **표 행(`| 위치 |`)** 형태로 갱신. 1:1 정합·필수필드(위치·현황·문제점·PM질문) 비어있지 않음 검사 의미는 유지, 구문 형태만 표 행으로. `처분`은 표 밖 라인으로 존재(비어도 무방)임을 명시.
- 그 외 체크(C5 제외/오판정 메인 재등장 금지, C7 🚫↔결번색인)는 처분 라인이 블록 내(표 밖)에 그대로 있으므로 변경 불필요.

### 3.2 인스턴스 retrofit (2파일)

대상: `차별화리뷰Pro-추천인코드/doc/`

**`GAP-*.md`**
- 헤더 메타 `대상 PRD` 5줄 base URL에 슬러그 복원 + 앵커 취약성 캡션 한 줄.
- 색인 표 24행 위치 셀 → 딥링크 앵커.
- 상세블록 24개 → 하이브리드 표 + 처분 분리줄.

**`TC-*.md`**
- 헤더 메타 5줄 base URL 슬러그 복원.
- 본문 위치 하이퍼링크 3개(US07-T06 1개·US04-T04 2개) → 딥링크. 표 셀 내 비링크 토큰(S1-B 등 문서내 상호참조)은 linkify하지 않음.

**앵커 도출 데이터:** retrofit 앵커는 추측 금지 — 5페이지 전체를 MCP 페치해 실제 헤딩 텍스트에서 규칙으로 도출. (L2=2082799647 이미 확보, L1·L3화면·L3BO·L4 4페이지 추가 페치.) 슬러그는 인스턴스 CLAUDE.md "대상 PRD" URL 그대로 사용.
