## 0. 공통(베이스 엔티티) 컬럼

대부분의 엔티티는 `BaseEntity` 또는 도메인별 `AdminBaseEntity`를 상속한다. 아래 공통 컬럼은 표에 명시하지 않는다.

| 컬럼 | 타입 | 설명 |
|---|---|---|
| `created_at` | DATETIME | 생성 일시 |
| `created_by` | BIGINT | 생성자 ID |
| `updated_at` | DATETIME | 수정 일시 |
| `updated_by` | BIGINT | 수정자 ID |
| `deleted_yn` 또는 `is_deleted` | CHAR(1) Y/N 또는 TINYINT(1) | soft delete 플래그 — 도메인별로 컬럼명이 다름 |
| `deleted_at` | DATETIME (nullable) | soft delete 시각 — `deleted_at IS NULL` 패턴을 쓰는 도메인은 이 컬럼만으로 삭제 판정 |

- `product-domain` 계열: `deleted_yn` (`@Where(clause = "deleted_yn = 'N'")`)
- `user-domain` 정회원/소셜: `deleted_yn` (Y/N), 어드민/공통코드: `is_deleted` (Boolean)
- `order-domain` 계열: 도메인별 혼재 (`is_deleted` / `deleted_at IS NULL` / `status = 'DELETE'` 등). 각 테이블 설명에서 명시.
- `BaseManualDateEntity` (product-domain): `created_at`을 자동 세팅하지 않고 수동 입력 (엑셀 일괄등록용).

## 1. 사용자/회원 (user-domain)

### 1.1 사용자(고객)

#### `user`
- **용도**: 정회원(사업자) 마스터 — 차별화상회의 핵심 고객 엔티티
- **설명**: 사업자등록증 기반 회원 정보(상호, 사업자번호)와 ERP 코드, 회원 상태/추가상태, 마케팅 수신동의(알림톡/이메일/앱푸시)를 관리. `base_user`와 1:1 (`base_user_id`가 PK이자 FK). `@Where deleted_yn='N'` soft delete.
- **주요 컬럼**:

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `base_user_id` | BIGINT (PK, FK→base_user.id) | 정회원 식별자 |
  | `name` | VARCHAR (개인정보) | 이름 |
  | `email` | VARCHAR (개인정보) | 이메일 |
  | `status` | VARCHAR (enum `UserStatusType`) | 회원 상태 (NORMAL/WITHDRAW 등) |
  | `additional_status` | VARCHAR (enum `UserAdditionalStatusType`) | 회원 상태 추가 정보 |
  | `user_type` | VARCHAR (enum `UserType`) | 회원 유형 |
  | `business_registration_no` | VARCHAR (개인정보, UQ) | 사업자등록번호 |
  | `tax_registration_id` | VARCHAR(4) | 종사업자 번호 |
  | `company_name` | VARCHAR | 상호명 |
  | `erp_code` | VARCHAR(50) (idx) | 현재 ERP 고객 코드 |
  | `parcel_erp_code` | VARCHAR(50) | 택배 ERP 코드 |
  | `direct_erp_code` | VARCHAR(50) | 직배 ERP 코드 |
  | `referral_code` | VARCHAR | 추천 코드 |
  | `business_verified_at` | DATETIME | 사업자 승인일시 (NULL=미승인) |
  | `platform_type` | VARCHAR (enum `PlatformType`) | 가입 플랫폼 (WEB/AOS/IOS) |
  | `recommender_username` | VARCHAR | 추천인 username |
  | `marketing_alimtalk_agreement_yn` | CHAR(1) Y/N | 알림톡 수신동의 |
  | `marketing_email_agreement_yn` | CHAR(1) Y/N | 이메일 수신동의 |
  | `marketing_app_push_agreement_yn` | CHAR(1) Y/N | 앱푸시 수신동의 |
  | `store_phone_no` | VARCHAR (개인정보) | 사업장 전화번호 |
  | `password_updated_at` | DATETIME | 비밀번호 수정일 |
  | `purchase_limited_at` | DATETIME | 구매제한 설정일 |

#### `base_user`
- **용도**: 모든 회원(정회원/가회원/관리자/소셜)의 공통 인증 마스터
- **설명**: 로그인 ID/비밀번호(암호화)/휴대폰번호/Role을 보관. `user` / `temp_user` / `admin_user`와 1:1. `@Where deleted_yn='N'` soft delete + `@DynamicUpdate`.
- **주요 컬럼**:

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `id` | BIGINT (PK) | |
  | `username` | VARCHAR (UQ) | 로그인 아이디 |
  | `password` | VARCHAR (PasswordEncoder 인코딩) | 암호화된 비밀번호 |
  | `mobile_phone_no` | VARCHAR (개인정보) | 휴대전화번호 |
  | `role_type` | VARCHAR (enum `RoleType`) | TEMP_USER / USER / ADMIN / SOCIAL_USER |
  | `app_push_allowed_yn` | CHAR(1) Y/N | 앱푸시 허용 여부 |
  | `tester_yn` | CHAR(1) Y/N | 테스터 여부 |
  | `last_accessed_at` | DATETIME | 마지막 접근 일시 |

#### `user_read_only`
- **용도**: `user` 테이블의 읽기 전용 뷰 엔티티 (탈퇴자 포함 조회용)
- **설명**: `@Immutable + @Subselect("SELECT * FROM user")`. 멤버십 어드민 조회 등 `@Where` 우회 케이스에서 사용. **실제 별도 테이블이 아닌 가상 엔티티**.
- **주요 컬럼**: `user`와 동일 (부분집합)

#### `user_archive`
- **용도**: 탈퇴 시점의 회원 식별/사업자 정보 아카이브
- **설명**: 탈퇴 처리 시 마스킹/덮어쓰기 전에 식별 정보를 보존.

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `base_user_id` | BIGINT (PK) | 탈퇴한 회원 ID |
  | `username` | VARCHAR | 로그인 아이디 (아카이브) |
  | `mobile_phone_no` | VARCHAR (개인정보) | 휴대전화번호 |
  | `name` | VARCHAR (개인정보) | 이름 |
  | `email` | VARCHAR (개인정보) | 이메일 |
  | `business_registration_no` | VARCHAR (개인정보) | 사업자등록번호 |
  | `tax_registration_id` | VARCHAR(4) | 종사업자 번호 |

#### `user_withdrawal`
- **용도**: 회원 탈퇴 신청/처리 상태 관리
- **설명**: 사유(다중) + 상세 사유 + 환불 계좌(Embedded)를 저장하고 어드민이 승인/반려. 다중 사유는 컬렉션 테이블 `withdrawal_reason_type`에 별도 저장.

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `id` | BIGINT (PK) | |
  | `user_id` | BIGINT (FK→base_user.id) | 탈퇴 신청자 |
  | `company_name` | VARCHAR | 상호명 |
  | `reason_detail` | VARCHAR | 탈퇴 상세 사유 |
  | `bank_account_info_*` | (Embedded) (개인정보) | 환불 계좌 정보 |
  | `status` | VARCHAR (enum `WithdrawalStatusType`) | PROGRESS / APPROVE / REJECT |

- **연관 테이블**: `withdrawal_reason_type` (`@ElementCollection`: id + `WithdrawalReasonType` enum)

#### `user_memo`
- **용도**: 어드민이 회원에 남기는 메모
- **설명**: `AdminBaseEntity` 상속, `@Where(is_deleted = false)` soft delete. TEXT 컬럼.

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `id` | BIGINT (PK) | |
  | `user_id` | BIGINT (idx) | 대상 회원 |
  | `memo` | TEXT | 메모 내용 |

#### `user_shipping`
- **용도**: 회원 배송지(택배/직배 공통)
- **설명**: `user`와 1:1. 배송 유형/주소/진입/출입/보관 방법.

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `id` | BIGINT (PK) | |
  | `user_id` | BIGINT (FK→user.base_user_id) | 회원 |
  | `shipping_type` | VARCHAR (enum `UserShippingType`) | PARCEL / DIRECT |
  | `zip_code` | VARCHAR (개인정보) | 우편번호 |
  | `address` | VARCHAR (개인정보) | 주소 |
  | `address_detail` | VARCHAR (개인정보) | 상세 주소 |
  | `entrance_method` | VARCHAR | 진입 방법 |
  | `door_access_method` | VARCHAR | 출입 방법 |
  | `storage_location` | VARCHAR | 보관 장소 |

- **설명**: `base_user` ↔ `terms` N:M. 가입/추가 동의 시 1행씩 INSERT.

  | 컬럼 | 타입 | 설명 |

- **용도**: 회원-메뉴 매핑

## 8. 알림 (infrastructure)

#### `notification`
- **용도**: 인앱 알림함 메시지 본문
- **설명**: 알림 제목/내용과 타겟 타입을 저장하고, `notification_target`과 1:N으로 수신자 목록을 가진다.

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `id` | BIGINT (PK) | |
  | `title` | VARCHAR | 알림 제목 |
  | `content` | VARCHAR | 알림 내용 |
  | `target_type` | VARCHAR (enum `NotificationType`) | 알림 타입 |

#### `notification_target`
- **용도**: 알림 수신자별 읽음 상태
- **설명**: 알림(`notification_id`)과 회원(`user_id`)을 매핑하고 읽음 여부를 관리. `user_id` 인덱스로 회원별 알림함 조회 최적화.

  | 컬럼 | 타입 | 설명 |
  |---|---|---|
  | `id` | BIGINT (PK) | |
  | `user_id` | BIGINT (idx) | 수신 회원 ID |
  | `notification_id` | BIGINT (FK) | `notification.id` 참조 |
  | `is_read` | BIT Y/N | 읽음 여부 |

## 부록: 참고 사항

- **PK 타입 다양성**: 대부분 BIGINT IDENTITY지만 일부는 VARCHAR(ULID), 이벤트 코드는 VARCHAR(우편번호), 멀티 컬럼 EmbeddedId 등 다양함.
- **soft delete 컬럼 다양성**: 도메인별/엔티티별로 `deleted_yn`/`is_deleted`/`deleted_at IS NULL`/`status='DELETE'` 등 혼재 — ETL에서 필터링 시 도메인별 규칙 확인 필수.
- **백틱 이스케이프 테이블**: `` `order` ``는 MySQL 예약어를 피하기 위해 백틱 이스케이프됨. 쿼리 작성 시 주의.
- **테이블명 오타**: `notice_popop_business_type`, `notice_popop_menu`는 운영 테이블명에 오타 `popop`이 그대로 사용 중.
- **엔티티 파일명 오타**: `PaymenetPgCancel.kt` (Payment 오타) — 테이블명은 정상 `payment_pg_cancel`.
- **Audit/Envers**: `product`, `food_product`, `product_discount`, `product_purchase_limit`, `product_sales_limit`, `direct_shipping*` 등은 Hibernate Envers 변경이력 감사 사용 (별도 `_AUD` 테이블 자동 생성).
- **민감 컬럼 ETL 가이드**: PG/결제(`payment_pg.pg_response`, `payment_batch_card.*key*`/`*token*`, `payment_virtual_account.*`, `refund_account.bank_account_info_*`)와 개인정보(`user.*`, `social_user.*`, `mobile_phone_authentication.mobile_phone_no` 등)는 데이터 마트 적재 시 마스킹/제외 정책을 적용해야 함.
- **멤버십(membership_*) 테이블**: 본 문서에서 의도적으로 제외. 필요 시 별도 문서로 정리.
